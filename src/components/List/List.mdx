# List

A flexible, accessible component for displaying vertical collections of items, supporting icons, headers, footers, dividers, and nested groups.

## Overview

The List component provides a consistent way to display ordered and unordered lists, with support for custom icons, section headers/footers, dividers, and nested groups. It is highly customizable and accessible.

## Basic Usage

```jsx
import { List, ListItem } from '@edu-ux/components';

<List>
  <ListItem>First item</ListItem>
  <ListItem>Second item</ListItem>
  <ListItem>Third item</ListItem>
</List>
```

## Ordered List

```jsx
<List as="ol">
  <ListItem>Step one</ListItem>
  <ListItem>Step two</ListItem>
  <ListItem>Step three</ListItem>
</List>
```


## API Reference

<Markdown>{`
| Prop | Type | Default | Description |
|------|------|---------|-------------|
|  children |  ReactNode | - | List content (required) |
|  as |  'ul' | 'ol' | 'ul' | List type (unordered or ordered) |
|  variant |  string | 'plain' | Visual variant (plain, bordered, inset, elevated) |
|  size |  string | 'md' | Size/density (sm, md, lg) |
|  divided |  boolean | false | Show dividers between items |
|  nested |  boolean | false | Is this a nested list? |
|  className |  string | - | Additional CSS classes |
|  style |  object | - | Inline styles |
`}</Markdown>

### Atomic Subcomponents

- `ListItem` — A single item in the list
- `ListIcon` — Icon for use in list items
- `ListGroup` — Grouping for nested or sectioned lists
- `ListDivider` — Visual divider between list sections
- `ListHeader` — Section header for a list
- `ListFooter` — Section footer for a list

## Best Practices

- Use `List` for any vertical collection of related items
- Use `ListItem` for each entry; use `icon` for visual context
- Use `ListHeader` and `ListFooter` to organize sections
- Use `ListGroup` for nested lists or subcategories
- Use `divided` for clear separation in dense lists
- Prefer semantic `ul`/`ol` for accessibility
- Keep list content concise and scannable

## Accessibility

- List uses semantic `ul`/`ol` elements by default
- All items are keyboard accessible
- Use headers and footers for screen reader context
- Dividers use `role="separator"`
- Nested lists are properly indented and grouped

## Migration Guide

If migrating from a previous List implementation:
- Use `List` instead of plain `ul`/`ol` for consistent styling
- Replace custom icons with `ListIcon` or the `icon` prop
- Use atomic subcomponents for advanced layouts
