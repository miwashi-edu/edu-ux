# Tabs

A flexible tabs component that organizes content into multiple sections, supporting various visual styles and orientations.

## Overview

The Tabs component provides a clean way to organize and display related content in separate sections. It supports multiple visual variants, sizes, and orientations while maintaining full accessibility compliance.

## Features

- **Multiple variants**: Default, bordered, pills, and underline styles
- **Flexible orientation**: Horizontal and vertical layouts
- **Accessibility**: Full ARIA support and keyboard navigation
- **Controlled state**: Support for external state management
- **Responsive design**: Adapts to different screen sizes
- **Compound components**: Flexible composition pattern

## Usage

```jsx
import Tabs from './components/Tabs';

// Basic usage
<Tabs defaultValue="account">
  <Tabs.List>
    <Tabs.Trigger value="account">Account</Tabs.Trigger>
    <Tabs.Trigger value="password">Password</Tabs.Trigger>
    <Tabs.Trigger value="settings">Settings</Tabs.Trigger>
  </Tabs.List>
  <Tabs.Content value="account">
    <div>Account settings content</div>
  </Tabs.Content>
  <Tabs.Content value="password">
    <div>Password settings content</div>
  </Tabs.Content>
  <Tabs.Content value="settings">
    <div>General settings content</div>
  </Tabs.Content>
</Tabs>

// Controlled usage
<Tabs value={activeTab} onValueChange={setActiveTab}>
  {/* Tab content */}
</Tabs>
```

## Accessibility

- Uses proper ARIA attributes (`role="tablist"`, `role="tab"`, `role="tabpanel"`)
- Implements `aria-selected` and `aria-disabled` on triggers
- Supports keyboard navigation (Enter, Space, Arrow keys)
- Focus management with proper tabindex values
- Screen reader friendly with descriptive labels
- High contrast mode support
- Reduced motion support for animations

## Best Practices

1. **Use descriptive labels**: Make tab labels clear and concise
2. **Limit tab count**: Avoid too many tabs to prevent cognitive overload
3. **Group related content**: Keep related information in the same tab
4. **Consider mobile**: Use appropriate orientation for mobile devices
5. **Provide fallbacks**: Ensure content is accessible even without JavaScript
6. **Test keyboard navigation**: Verify all interactions work without a mouse
