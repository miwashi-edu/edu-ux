# IconButton

A button component that displays only an icon, perfect for compact actions and toolbar interfaces.

## Overview

The IconButton component provides a clean, accessible way to create icon-only buttons. It supports multiple variants, sizes, and states including loading and disabled states.

## Features

- **Multiple variants**: Default, primary, secondary, success, warning, error, and ghost styles
- **Three sizes**: Small, medium, and large
- **Interactive states**: Loading spinner and disabled states
- **Accessibility**: ARIA labels and keyboard navigation
- **Responsive design**: Adapts to different screen sizes

## Usage

```jsx
import IconButton from './components/IconButton';

// Basic usage
<IconButton 
  icon={<HeartIcon />} 
  onClick={handleLike}
  aria-label="Like post"
/>

// With variant and size
<IconButton 
  icon={<EditIcon />}
  variant="primary"
  size="large"
  onClick={handleEdit}
/>

// Loading state
<IconButton 
  icon={<SaveIcon />}
  loading={true}
  onClick={handleSave}
/>
```

## Accessibility

- Always provide `aria-label` for screen reader support
- Implements proper focus management
- Supports keyboard navigation
- Loading state shows animated spinner
- High contrast mode support
- Reduced motion support for animations

## Best Practices

1. **Always include aria-label**: Icon-only buttons need descriptive labels for accessibility
2. **Choose appropriate variants**: Use variants that match the action's importance
3. **Consider loading states**: Show loading feedback for async actions
4. **Use consistent sizing**: Maintain consistent button sizes within interfaces
5. **Test with screen readers**: Ensure buttons are properly announced
