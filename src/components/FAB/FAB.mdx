# FAB

A Floating Action Button (FAB) component that provides quick access to primary actions, typically positioned in a fixed location on the screen.

## Overview

The FAB component is a circular button that floats above the content, usually containing the primary action for a screen. It's commonly used in mobile-first designs and follows Material Design principles.

## Features

- **Multiple variants**: Primary, secondary, success, warning, and error styles
- **Three sizes**: Small, medium, and large
- **Flexible positioning**: Five different position options
- **Hover labels**: Optional labels that appear on hover
- **Accessibility**: ARIA labels and keyboard navigation
- **Responsive design**: Adapts to different screen sizes

## Usage

```jsx
import FAB from './components/FAB';

// Basic usage
<FAB 
  icon={<PlusIcon />} 
  label="Add new item"
  onClick={handleAdd}
/>

// With custom position and variant
<FAB 
  icon={<EditIcon />}
  label="Edit"
  variant="secondary"
  position="top-right"
  size="large"
  onClick={handleEdit}
/>
```

## Accessibility

- Uses `aria-label` for screen reader support
- Implements proper focus management
- Supports keyboard navigation
- High contrast mode support
- Reduced motion support for animations

## Best Practices

1. **Use for primary actions**: Reserve FABs for the most important action on a screen
2. **Provide clear labels**: Always include descriptive labels for accessibility
3. **Choose appropriate positioning**: Consider the user's thumb reach on mobile devices
4. **Limit to one per screen**: Avoid multiple FABs that could confuse users
5. **Use consistent icons**: Choose icons that clearly represent the action
