# Header

A page-level header component that provides a flexible and accessible way to create site headers with support for branding, navigation, and action areas.

## Overview

The Header component is designed to be the topmost section of a web page, typically containing branding, navigation, and user actions. It supports various layouts, responsive behavior, and can include custom content and actions.

## Features

- **Responsive Design**: Automatically adapts to different screen sizes
- **Multiple Variants**: Default, minimal, bordered, and elevated styles
- **Flexible Layout**: Support for brand, content, and action areas
- **Fixed Positioning**: Option to fix header to the top of the viewport
- **Transparent Mode**: Support for transparent backgrounds
- **Accessibility**: Built with ARIA attributes and keyboard navigation
- **Dark Mode Support**: Theme-aware styling
- **Customizable**: Extensive prop options for customization

## Variants

The Header component supports four visual variants:

### Default
The standard header with background color and subtle shadow.

### Minimal
A clean, minimal design with reduced visual weight.

### Bordered
A bordered variant with a border around the header.

### Elevated
An elevated variant with a more prominent shadow effect.

## Sizes

Three size options are available:

- **Small (sm)**: Compact header for space-constrained layouts
- **Medium (md)**: Standard size for most applications
- **Large (lg)**: Larger header for prominent branding areas

## Responsive Behavior

The Header automatically adapts to different screen sizes:

- **Desktop**: Full header with all content visible
- **Tablet**: Responsive layout with potential content wrapping
- **Mobile**: Compact layout with mobile-optimized interactions

## Accessibility

The Header component includes several accessibility features:

- Proper ARIA labels and roles
- Keyboard navigation support
- Focus management
- Screen reader compatibility
- High contrast support

## Best Practices

1. **Clear Branding**: Make sure your brand is clearly visible
2. **Logical Layout**: Arrange content in a logical order
3. **Responsive Design**: Test on different screen sizes
4. **Accessibility**: Ensure keyboard navigation works properly
5. **Performance**: Keep header content lightweight
6. **Consistency**: Use consistent styling across your application
