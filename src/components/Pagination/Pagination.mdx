# Pagination

A pagination component that provides navigation through multiple pages of content with customizable controls and styling.

## Overview

The Pagination component offers a complete solution for navigating through paginated content. It supports various configurations including first/last page buttons, previous/next navigation, and customizable page ranges.

## Features

- **Multiple variants**: Default, bordered, and rounded styles
- **Three sizes**: Small, medium, and large
- **Flexible controls**: Optional first/last and previous/next buttons
- **Smart page range**: Configurable maximum visible pages
- **Accessibility**: ARIA labels and keyboard navigation
- **Responsive design**: Adapts to different screen sizes

## Usage

```jsx
import Pagination from './components/Pagination';

// Basic usage
<Pagination 
  currentPage={1}
  totalPages={10}
  onPageChange={(page) => console.log('Page changed to:', page)}
/>

// Custom configuration
<Pagination 
  currentPage={3}
  totalPages={15}
  showFirstLast={false}
  showPrevNext={true}
  maxVisible={7}
  variant="rounded"
  size="large"
  onPageChange={handlePageChange}
/>
```

## Accessibility

- Uses semantic `<nav>` element with proper `aria-label`
- Implements `aria-current="page"` for the active page
- Provides descriptive `aria-label` for navigation buttons
- Supports keyboard navigation
- High contrast mode support
- Reduced motion support for animations

## Best Practices

1. **Choose appropriate size**: Use larger sizes for touch interfaces
2. **Limit visible pages**: Keep `maxVisible` reasonable to avoid cluttered UI
3. **Provide clear feedback**: Show current page and total pages when possible
4. **Handle edge cases**: Ensure pagination works with single page or empty results
5. **Test keyboard navigation**: Ensure all buttons are accessible via keyboard
