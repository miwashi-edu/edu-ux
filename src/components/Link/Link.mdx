# Link

A versatile link component that provides consistent styling and behavior for navigation and external links.

## Overview

The Link component provides a standardized way to create links with various visual styles, sizes, and behaviors. It supports external links with automatic icon indicators and proper security attributes.

## Features

- **Multiple variants**: Default, primary, secondary, success, warning, and error styles
- **Three sizes**: Small, medium, and large
- **External link support**: Automatic icon and security attributes
- **Disabled state**: Prevent interaction when needed
- **Accessibility**: ARIA attributes and keyboard navigation
- **Responsive design**: Adapts to different screen sizes

## Usage

```jsx
import Link from './components/Link';

// Basic usage
<Link href="/about" variant="primary">
  About Us
</Link>

// External link
<Link 
  href="https://example.com" 
  external={true}
  variant="primary"
>
  Visit External Site
</Link>

// Disabled link
<Link 
  href="/disabled" 
  disabled={true}
  variant="secondary"
>
  Disabled Link
</Link>
```

## Accessibility

- Uses semantic `<a>` element for proper screen reader support
- Implements proper focus management with visible focus indicators
- External links include `rel="noopener noreferrer"` for security
- Disabled links are properly marked with `aria-disabled`
- High contrast mode support
- Reduced motion support for animations

## Best Practices

1. **Use descriptive text**: Make link text clear and descriptive
2. **Choose appropriate variants**: Use variants that match the link's purpose
3. **Mark external links**: Always use the `external` prop for external links
4. **Handle disabled state**: Provide clear feedback when links are disabled
5. **Test keyboard navigation**: Ensure all links are accessible via keyboard
