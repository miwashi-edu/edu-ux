# Tooltip

A flexible tooltip component that displays additional information when users hover over or focus on elements, supporting multiple positions and visual variants.

## Overview

The Tooltip component provides contextual information to users through hover or focus interactions. It supports various positions, visual styles, and content types while maintaining full accessibility compliance.

## Features

- **Multiple positions**: Eight different positioning options
- **Visual variants**: Default, info, success, warning, and error styles
- **Flexible sizing**: Small, medium, and large sizes
- **Customizable delay**: Configurable show/hide timing
- **Accessibility**: Full ARIA support and keyboard navigation
- **Responsive design**: Adapts to different screen sizes
- **Viewport awareness**: Automatically adjusts position to stay in view

## Usage

```jsx
import Tooltip from './components/Tooltip';

// Basic usage
<Tooltip content="This is a helpful tooltip">
  <Button>Hover me</Button>
</Tooltip>

// With custom position and variant
<Tooltip 
  content="Informational tooltip" 
  position="right" 
  variant="info"
>
  <span>Help text</span>
</Tooltip>

// With custom delay
<Tooltip content="Delayed tooltip" delay={500}>
  <Button>Delayed</Button>
</Tooltip>
```

## Accessibility

- Uses `role="tooltip"` for proper semantic meaning
- Supports keyboard navigation with focus/blur events
- Implements proper ARIA attributes
- Screen reader friendly with descriptive content
- High contrast mode support
- Reduced motion support for animations
- Viewport-aware positioning to ensure visibility

## Best Practices

1. **Keep content concise**: Tooltips should be brief and to the point
2. **Use appropriate variants**: Match the tooltip style to the content type
3. **Consider positioning**: Choose positions that don't interfere with other UI elements
4. **Test keyboard navigation**: Ensure tooltips work with keyboard focus
5. **Avoid overuse**: Don't add tooltips to every element
6. **Provide meaningful content**: Tooltips should add value, not just repeat visible text

## Content Types

Tooltips support various content types:

```jsx
// Simple text
<Tooltip content="Simple tooltip text">
  <Button>Text</Button>
</Tooltip>

// Rich content with formatting
<Tooltip content={
  <div>
    <strong>Bold text</strong>
    <br />
    With line breaks
  </div>
}>
  <Button>Rich</Button>
</Tooltip>

// Complex structured content
<Tooltip content={
  <div>
    <div style={{ fontWeight: 'bold', marginBottom: '0.25rem' }}>Title</div>
    <div style={{ fontSize: '0.75rem', opacity: 0.9 }}>
      Detailed description with multiple elements
    </div>
  </div>
}>
  <Button>Complex</Button>
</Tooltip>
```
