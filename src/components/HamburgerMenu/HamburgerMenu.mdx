# HamburgerMenu

A mobile-friendly navigation component that provides a collapsible menu triggered by a hamburger icon, perfect for responsive navigation on smaller screens.

## Overview

The HamburgerMenu component is designed to provide mobile-friendly navigation with a collapsible menu triggered by a hamburger icon. It's perfect for responsive designs where space is limited and traditional navigation needs to be hidden.

## Features

- **Mobile-First Design**: Optimized for mobile and tablet devices
- **Collapsible Menu**: Expandable navigation with smooth animations
- **Multiple Animations**: Slide, fade, and scale animation options
- **Multiple Variants**: Default, minimal, bordered, and elevated styles
- **Responsive Design**: Automatically adapts to different screen sizes
- **Accessibility**: Built with ARIA attributes and keyboard navigation
- **Dark Mode Support**: Theme-aware styling
- **Customizable**: Extensive prop options for customization

## Usage

```jsx
import HamburgerMenu from './components/HamburgerMenu';

function App() {
  return (
    <HamburgerMenu
      items={[
        { id: 'home', label: 'Home', href: '/', icon: '🏠' },
        { id: 'about', label: 'About', href: '/about', icon: 'ℹ️' },
        { id: 'services', label: 'Services', href: '/services', icon: '🛠️' },
        { id: 'contact', label: 'Contact', href: '/contact', icon: '📞' }
      ]}
      variant="default"
      size="md"
      animation="slide"
    />
  );
}
```

## Props

<Markdown>{`
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| items | array | [] | Navigation items array |
| items[].id | string | - | Unique identifier for the item |
| items[].label | string | - | Display text for the item |
| items[].href | string | - | Link URL for the item |
| items[].icon | string | - | Icon for the item (emoji or icon component) |
| sections | array | [] | Navigation sections array |
| sections[].id | string | - | Unique identifier for the section |
| sections[].title | string | - | Section title |
| sections[].items | array | - | Items in the section |
| trigger | node | - | Custom trigger element |
| variant | string | 'default' | Visual variant: 'default', 'minimal', 'bordered', 'elevated' |
| size | string | 'md' | Size: 'sm', 'md', 'lg' |
| animation | string | 'slide' | Animation: 'slide', 'fade', 'scale' |
| header | node | - | Optional header content |
| footer | node | - | Optional footer content |
| className | string | '' | Additional CSS classes |

## Variants

The HamburgerMenu component supports four visual variants:

### Default
The standard hamburger menu with background color and subtle shadow.

### Minimal
A clean, minimal design with reduced visual weight.

### Bordered
A bordered variant with a border around the menu.

### Elevated
An elevated variant with a more prominent shadow effect.

## Sizes

Three size options are available:

- **Small (sm)**: Compact hamburger menu for space-constrained layouts
- **Medium (md)**: Standard size for most applications
- **Large (lg)**: Larger hamburger menu for prominent navigation areas

## Animations

Three animation types are available:

- **Slide**: Smooth sliding animation (default)
- **Fade**: Fade in/out animation
- **Scale**: Scale in/out animation

## Responsive Behavior

The HamburgerMenu automatically adapts to different screen sizes:

- **Desktop**: Can be used as an alternative navigation method
- **Tablet**: Responsive layout with touch-friendly interactions
- **Mobile**: Optimized for mobile devices with full-screen overlay

## Accessibility

The HamburgerMenu component includes several accessibility features:

- Proper ARIA labels and roles
- Keyboard navigation support
- Focus management
- Screen reader compatibility
- High contrast support
- Reduced motion support

## Best Practices

1. **Mobile-First**: Design primarily for mobile devices
2. **Clear Labels**: Use descriptive labels for navigation items
3. **Logical Order**: Arrange items in a logical order
4. **Performance**: Keep the menu lightweight for better performance
5. **Accessibility**: Test keyboard navigation thoroughly
6. **Touch-Friendly**: Ensure adequate touch targets for mobile
