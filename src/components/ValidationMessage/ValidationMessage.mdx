# ValidationMessage

A component for displaying validation messages with different states and types.

## Overview

The ValidationMessage component provides a consistent way to display feedback to users about form validation, operation results, or system status. It supports multiple message types with appropriate styling and accessibility features.

## Features

- **Multiple validation states**: Info, success, warning, and error
- **Accessibility**: ARIA attributes and screen reader support
- **Responsive design**: Adapts to different screen sizes
- **Customizable**: Supports custom CSS classes
- **Conditional rendering**: Can be shown/hidden based on conditions

## Usage

```jsx
import ValidationMessage from './components/ValidationMessage';

// Basic usage
<ValidationMessage 
  message="Please enter a valid email address"
  type="error"
/>

// Success message
<ValidationMessage 
  message="Your changes have been saved successfully!"
  type="success"
/>

// Conditional rendering
<ValidationMessage 
  message={errorMessage}
  type="error"
  show={!!errorMessage}
/>
```

## Accessibility

- Uses `role="alert"` for important messages
- Implements `aria-live="polite"` for screen reader announcements
- Provides high contrast mode support
- Supports reduced motion preferences
- Includes focus indicators for keyboard navigation

## Best Practices

1. **Use appropriate message types**: Choose the correct type based on the message content
2. **Keep messages concise**: Write clear, actionable messages
3. **Provide context**: Explain what the user should do next
4. **Use conditional rendering**: Only show messages when relevant
5. **Test with screen readers**: Ensure messages are properly announced
